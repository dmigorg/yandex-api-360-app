# Правила обработки писем

## Обзор

Правила обработки писем представляют собой инструмент для управления потоками входящей и исходящей почты. Администраторы могут создавать правила, которые действуют на уровне домена и имеют приоритет над аналогичными правилами, установленными пользователями.

## Разрешения на использование сервиса

- `ya360_admin:mail_write_routing_rules` — для просмотра и изменения
- `ya360_admin:mail_read_routing_rules` — для просмотра

## Терминология

**Правило** — инструкция, описывающая какое действие выполнить с письмом при выполнении определенного условия.

**Действие** — операция над письмом. Возможные значения:
- drop — не доставлять письмо
- forward — переслать письмо

**Условие** — алгоритм проверки письма на соответствие определенным критериям.

## Порядок работы

Правила применяются либо к входящей, либо к исходящей почте в зависимости от параметра `direction`. Каждое письмо проверяется на соответствие условиям (например, по содержанию темы, заголовков, вложений, адресу отправителя). При выполнении условия выполняется соответствующее действие.

## Особенности

Правила распространяются только на сотрудников с аккаунтами на домене организации. На пользователей с внешними аккаунтами (включая владельцев организаций с почтой типа `@yandex.ru`) правила не действуют.

## Примеры применения

**Входящая почта:**
- Пересылка всей входящей корреспонденции одного сотрудника другому
- Установка запрета на отправку писем на определенный адрес для всех, кроме конкретных пользователей

**Исходящая почта:**
- Блокировка писем на конкретные адреса
- Пересылка писем с определенной темой на заданный адрес
