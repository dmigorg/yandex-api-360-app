# Разрешенные и запрещенные отправители

Сервис для настройки на уровне домена разрешенных и запрещенных адресов отправителей входящей почты с помощью правил Domain Policy Rules. Эти правила имеют приоритет над настройками защиты от спама в Яндекс Почте.

## Разрешения на использование сервиса

- `ya360_admin:mail_write_routing_rules` — просмотр и изменение
- `ya360_admin:mail_read_routing_rules` — только просмотр

## Терминология

**Правило** — инструкция, описывающая какое действие выполнить с письмом при выполнении определенного условия. У каждого правила может быть название и описание.

**Действие** — операции над письмами:
- Reject — не доставлять независимо от обнаружения спама
- Accept with spam marking — помещать в папку «Спам» получателя
- Accept — доставлять, даже если помечено как спам

**Условие** — алгоритм проверки отправителя с использованием одного из списков:
- Полные адреса электронной почты
- Домены
- IP-адреса и маски подсетей

Максимум 200 элементов в списке.

## Порядок работы с правилами

- Список правил обновляется целиком; для добавления правил необходимо скачать, изменить и загрузить список заново
- Порядок правил важен — правила с более высоким приоритетом применяются первыми; обработка останавливается после первого совпадения
- Правила можно временно отключить через флаг `false` без удаления
- Правила применяются только к сотрудникам организации, не к внешним аккаунтам типа `login@yandex.ru`
- Максимум 200 правил в списке
