# Удалить подразделение

Удаляет подразделение. Запрос может быть выполнен только для подразделения без сотрудников и вложенных подразделений.

## Примечание

Требуется разрешение на изменение данных о подразделениях.

## Request

```
DELETE https://api360.yandex.net/directory/v1/org/{orgId}/departments/{departmentId}
```

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|---------|
| `departmentId` | integer | Идентификатор подразделения |
| `orgId` | integer | Идентификатор организации |

## Responses

### 200 OK

Запрос успешно выполнен.

```json
{
  "id": 0,
  "removed": true
}
```

| Параметр | Тип | Описание |
|----------|-----|---------|
| `id` | integer | Идентификатор подразделения |
| `removed` | boolean | Признак удаления: `true` — удалено; `false` — не удалено |

### 400 Bad Request

Некорректный запрос.

### 401 Unauthorized

Пользователь не авторизован.

### 403 Forbidden

У пользователя или приложения нет прав на доступ к ресурсу.

### 404 Not Found

Запрашиваемый ресурс не найден.

### 500 Internal Server Error

Внутренняя ошибка сервиса. Попробуйте повторно отправить запрос через некоторое время.
