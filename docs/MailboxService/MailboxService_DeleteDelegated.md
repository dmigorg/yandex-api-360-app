# Выключить для ящика возможность делегирования

Запрещает выдачу доступа сотрудников к ящику.

Идентификатор `resourceId` обозначает почтовый ящик, для которого отключается возможность делегирования. Для делегированных ящиков идентификатор совпадает с идентификатором сотрудника-владельца.

## Примечание

Чтобы выполнить запрос, приложению требуется разрешение на управление правами доступа к почтовым ящикам. Убедитесь, что вы включили его при настройке приложения на OAuth-сервере Яндекса.

## Request

```
DELETE https://api360.yandex.net/admin/v1/org/{orgId}/mailboxes/delegated/{resourceId}
```

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|---------|
| `orgId` | integer | Идентификатор организации |
| `resourceId` | string (uint64) | Идентификатор почтового ящика |

## Responses

### 200 OK

Запрос успешно выполнен.

**Body:** `{}`

### 400 Bad Request

Некорректный запрос.

### 401 Unauthorized

Пользователь не авторизован.

### 403 Forbidden

У пользователя или приложения нет прав на доступ к ресурсу, запрос отклонен.

### 404 Not Found

Запрашиваемый ресурс не найден.

### 500 Internal Server Error

Внутренняя ошибка сервиса. Попробуйте повторно отправить запрос через некоторое время.

Все ответы об ошибках содержат:
- `code` (integer): Код ошибки
- `message` (string): Описание ошибки
- `details` (array): Дополнительная информация об ошибке
