# Создать правило автоответа или пересылки

Создает правило автоответа или пересылки писем для сотрудника организации. Пересылка доступна только на домены, принадлежащие выбранной организации, и не требует подтверждения получения.

## Примечание

Приложению необходимо разрешение на управление настройками почты пользователя при настройке на OAuth-сервере Яндекса.

## Request

```
POST https://api360.yandex.net/admin/v1/org/{orgId}/mail/users/{userId}/settings/user_rules
```

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|---------|
| `orgId` | integer | Идентификатор организации |
| `userId` | string (uint64) | Идентификатор пользователя |

### Body (application/json)

```json
{
  "autoreply": {
    "ruleName": "example",
    "text": "example"
  },
  "forward": {
    "ruleName": "example",
    "address": "example",
    "withStore": true
  }
}
```

#### Правило автоответа (autoreply)

| Параметр | Тип | Описание |
|----------|-----|---------|
| `ruleName` | string | Название правила |
| `text` | string | Текст автоответа |

#### Правило пересылки (forward)

| Параметр | Тип | Описание |
|----------|-----|---------|
| `ruleName` | string | Название правила |
| `address` | string | Email получателя пересылаемого письма |
| `withStore` | boolean | Сохранять копию письма в ящике исходного получателя |

## Responses

### 200 OK

Запрос успешно выполнен.

```json
{
  "ruleId": 0
}
```

| Параметр | Тип | Описание |
|----------|-----|---------|
| `ruleId` | integer | Идентификатор созданного правила |

### 400 Bad Request

Некорректный запрос.

### 401 Unauthorized

Пользователь не авторизован.

### 403 Forbidden

Отсутствуют права доступа.

### 404 Not Found

Ресурс не найден.

### 500 Internal Server Error

Внутренняя ошибка сервиса.
